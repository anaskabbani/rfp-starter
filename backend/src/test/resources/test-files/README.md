# Test Files

This directory contains sample files used for document extraction testing.

## Files Generated

All test files have been automatically generated by `TestFileGenerator.java`.

### 1. sample.pdf
- **Format**: PDF (2 pages)
- **Content**:
  - Page 1: RFP header with key-value pairs for extraction testing
    - Carrier: Aetna
    - Due Date: January 15, 2025
    - Contact: John Smith
    - Email: john.smith@example.com
    - Phone: (555) 123-4567
    - RFP Number: RFP-2025-001
  - Page 2: Coverage requirements with additional details
- **Purpose**: Tests PDF text extraction, page count, and key-value pair extraction

### 2. sample-with-tables.docx
- **Format**: DOCX (Microsoft Word)
- **Content**:
  - Title: "Insurance Proposal Summary"
  - Introductory paragraph
  - 4x4 table with plan comparison (Plan Type, Deductible, Monthly Premium, Coverage Level)
  - Closing paragraph
- **Purpose**: Tests DOCX extraction with both text and tables

### 3. sample-simple.docx
- **Format**: DOCX (Microsoft Word)
- **Content**:
  - Title: "Benefits Overview"
  - Multiple paragraphs with company info and benefits description
  - No tables
- **Purpose**: Tests DOCX extraction without tables (tableCount should be 0)

### 4. sample.xlsx
- **Format**: XLSX (Microsoft Excel)
- **Content**:
  - Sheet1: "Plan Details" - 5 rows with plan information (text, numbers, booleans)
  - Sheet2: "Pricing" - 5 rows with pricing tiers (text and decimal numbers)
- **Purpose**: Tests XLSX extraction with multiple sheets and mixed data types

## Regenerating Files

To regenerate these files, run:
```bash
mvn test-compile exec:java -Dexec.mainClass="com.acme.saas.util.TestFileGenerator" -Dexec.classpathScope=test
```

## Test Data Characteristics

All test data uses realistic insurance/RFP content to match the domain:
- Key-value pairs follow the pattern: `Key: Value` (for extraction testing)
- Tables contain structured plan and pricing information
- Multiple data types are used (text, numbers, booleans, decimals)
- Content is meaningful and represents actual business documents
